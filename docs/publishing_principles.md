# Publishing principles

## Goal

Prepare Markdown-based operational notes so they can be published to Confluence with minimal cleanup.

## Rules

- keep section order stable across documents
- use short operational bullets instead of long narratives
- separate decisions from discussions
- keep action items explicit and attributable
- preserve source links and evidence
- avoid note formats that depend on local editor features only

## Good outcomes

- meeting notes are easy to skim
- task systems can ingest follow-up actions
- stakeholders can find historical decisions quickly
- source Markdown remains useful before and after publication
